# UP-MVP

MVP модуля формирования учебного плана.

## Запуск с нуля

1. Создать и активировать виртуальное окружение Python.
2. Установить backend-зависимости из `requirements.txt`.
3. Убедиться, что локально установлен и запущен Ollama.
4. Установить frontend-зависимости в `frontend/`.
5. Запустить backend на `http://127.0.0.1:8000`.
6. Запустить frontend на `http://127.0.0.1:5173`.

## Backend

Требования:
- Python 3.11+

Установка зависимостей:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Запуск backend:

```bash
uvicorn backend.main:app --reload
```

Проверка health-check:

```bash
curl http://127.0.0.1:8000/api/v1/health
```

Ожидаемый ответ:

```json
{"status":"ok"}
```

## Ollama

Проверка установки и доступных моделей:

```bash
ollama --version
ollama list
```

В текущем проекте по умолчанию используется модель `llama3:latest`.
Если хочешь использовать другую локальную модель, задай переменную окружения `OLLAMA_MODEL`.
При медленном ответе локальной модели можно увеличить таймаут через `OLLAMA_TIMEOUT`.

Пример для PowerShell:

```powershell
$env:OLLAMA_MODEL="llama3:latest"
$env:OLLAMA_TIMEOUT="180"
uvicorn backend.main:app --reload
```

Если Ollama уже запущен как Windows-приложение, отдельная команда `ollama serve` не нужна.

Быстрая проверка LLM через API проекта:

1. Запустить backend.
2. Создать план.
3. Добавить хотя бы один элемент в Таблицу 2.
4. Вызвать `POST /api/v1/plans/{plan_id}/validate`.
5. Убедиться, что в ответе заполнено поле `llm_recommendations`.

## Frontend

Установка зависимостей:

```bash
cd frontend
npm install
```

Запуск dev-сервера:

```bash
cd frontend
npm run dev
```

Frontend ожидает backend по адресу `http://localhost:8000` через proxy Vite.

Что уже есть в skeleton:
- React 18 + Vite;
- proxy `/api` -> `http://localhost:8000`;
- `src/api.js` с базовым axios-инстансом;
- страницы `Table1`, `Table2`, `Table3`;
- базовая проверка backend через `/api/v1/health`.

Что уже реализовано в рабочих экранах:
- список планов и создание нового плана;
- Таблица 1 с рекомендациями, manual mode для ПКС и переносом в Таблицу 2;
- Таблица 2 с добавлением, редактированием и удалением строк;
- Таблица 3 с агрегатами, нарушениями, LLM-рекомендациями, утверждением и экспортом XLSX.

## Демо-сценарий

1. Открыть frontend.
2. Создать новый план.
3. Перейти в Таблицу 1 и выполнить перенос рекомендаций в Таблицу 2.
4. В Таблице 2 при необходимости вручную добавить элементы для ПКС или недостающих блоков.
5. Перейти в Таблицу 3 и нажать «Проверить учебный план».
6. Посмотреть список нарушений и текст `llm_recommendations`.
7. Скачать XLSX.
8. Попробовать утвердить план.
9. Если есть критические ошибки или ошибки, утверждение будет заблокировано.
10. После исправления структуры плана повторно выполнить проверку и утвердить план.
